What It Is

A large-scale health data lake built on Microsoft SQL Server — developed in partnership with the University of Pécs (PTE). The cooperation started in 2017. The formal InnoHealth DataLake project launched in 2018 and has operated continuously since, with active sub-projects in oncology data mining and multimodal patient monitoring.

The platform was featured as a flagship customer case study at Microsoft's SQL Server 2019 Uncovered Virtual Event — selected by Microsoft to demonstrate what their platform could do at scale.

What It Does

The DataLake implements cognitive modelling and data processing methods for evaluating health data. The system includes:

  • Oncology indicator extraction from free-text medical records
  • Integration of laboratory test results, wearable device data (blood pressure, glucose, smart insoles, mattresses) and ECG monitors
  • Text classification using simple, complex, and advanced NLP models
  • Real-time reporting and BI visualisation via Power BI
  • Framework for data collection, representation, storage, recovery, management, and security
  • Unified standard and accreditation system for health IT

Sub-project: Multimodal Monitoring

Started in 2018 as a medical sub-project under the InnoHealth DataLake collaboration with PTE. The goal: a portable forecasting solution for patients with migraine and drug-resistant epilepsy that enables timely intervention before a seizure or attack.

  • Wearable IoT data streams — ECG (HR, HRV, QT interval), breathing (chest movement, rate, depth), accelerometer (motion and body position)
  • Automated data cleaning via machine learning with specifying markers, replacing lengthy manual data preparation
  • Analytical toolchain delivered as Python library and Knime module for further data science research
  • Prediction algorithm with an error rate below 1 per hour

DLX Architecture — Smart DataLake Platform

InnoHealth is built on E-Group's Smart DataLake Platform (DLX) — a 4-layer architecture designed to transform siloed data into actionable insights:

  • Data Layer — data injection, integration, structure, encryption, and virtualisation. Handles both horizontally scaled (large volumes) and vertically scaled (multiple types) structured and unstructured data
  • Analytical Layer — Business Intelligence (Power BI), ML/AI tools for root-cause analysis, prediction, and intervention timing
  • Knowledge & Research Layer — real-time reports, alerts, and integration with operational systems. Turns analytical results into actionable procedures
  • Unified Management Layer — system traceability, accessibility, and common operation services

The technology stack: SQL Server, MinIO, SPARK, and Kubernetes (K8S). Deployable in cloud or hybrid infrastructure with full data control, data privacy, elastic scalability, and a diverse set of analytic engines.

The platform adopts the OMOP Common Data Model — the international standard for harmonising observational health data — and was built from day one with GDPR-compliant privacy impact assessments governing every data flow.

Principal Architect

Alexandra Babaics — E-Group's Principal Architect for the InnoHealth DataLake. She presented the project at the Microsoft SQL Server 2019 Uncovered Virtual Event, demonstrating E-Group's capability in large-scale health data architecture.

EU Funding

The project was developed with EU funding through the Széchenyi 2020 programme, as part of a broader initiative to develop cognitive modelling and data processing methods for health data evaluation. The work established frameworks for data collection, representation, storage, and security that inform E-Group's broader health data work including FedX and HeliX.

From InnoHealth to FedX and HeliX

The InnoHealth DataLake is the direct architectural ancestor of E-Group's later federated health data work. Its federated extension — FDLX (Federated Data Lab) — extends the DLX concept across multiple institutions: analytics travel to the data, not the other way round. No central data storage, encrypted gradient sharing only, complete GDPR compliance.

The data modelling, health IT standards, and large-scale data architecture expertise developed through InnoHealth directly informed the design of the FedX federated learning platform and the HeliX health data space — now one of Europe's largest sovereign health data ecosystems. And while the platform was built for healthcare, the same architecture now underpins data-driven decision support in government, financial, and energy sectors.
